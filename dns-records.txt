CrimsonProject — GitHub Pages custom domain DNS setup
======================================================

Your live app URL (after DNS + Pages are configured):
  https://www.crimsonproject.com

Replace YOUR_GITHUB_USERNAME with your actual GitHub username everywhere below.


STEP 1 — Buy the domain
-----------------------
Register crimsonproject.com at any registrar, for example:
  - Cloudflare Registrar
  - Namecheap
  - Google Domains / Squarespace Domains
  - Porkbun

You must own crimsonproject.com before DNS can pass.


STEP 2 — Upload these files to your GitHub repo (root folder)
-------------------------------------------------------------
  index.html
  manifest.json
  icon.svg
  CNAME                  ← contains: www.crimsonproject.com
  .nojekyll


STEP 3 — Enable GitHub Pages
----------------------------
Repo → Settings → Pages
  Source: Deploy from a branch
  Branch: main
  Folder: / (root)

Then under "Custom domain", type:
  www.crimsonproject.com

Click Save. Wait for the DNS check (can take 5 min – 48 hrs after records propagate).


STEP 4 — DNS records at your registrar (passes GitHub check)
-----------------------------------------------------------

Record 1 — WWW (required)
  Type:  CNAME
  Name:  www
  Value: YOUR_GITHUB_USERNAME.github.io
  TTL:   Auto or 3600

Record 2 — Apex root domain (recommended so crimsonproject.com also works)
  Type:  A
  Name:  @
  Value: 185.199.108.153

  Type:  A
  Name:  @
  Value: 185.199.109.153

  Type:  A
  Name:  @
  Value: 185.199.110.153

  Type:  A
  Name:  @
  Value: 185.199.111.153

Optional IPv6 (GitHub recommends):
  Type:  AAAA
  Name:  @
  Value: 2606:50c0:8000::153

  Type:  AAAA
  Name:  @
  Value: 2606:50c0:8001::153

  Type:  AAAA
  Name:  @
  Value: 2606:50c0:8002::153

  Type:  AAAA
  Name:  @
  Value: 2606:50c0:8003::153

In GitHub Pages settings you can also add apex domain:
  crimsonproject.com
(GitHub will redirect www ↔ apex once both are verified.)


STEP 5 — Cloudflare users (if DNS is proxied)
---------------------------------------------
Set the orange cloud to DNS only (grey cloud) for:
  - www CNAME
  - apex A records

Or use Full (strict) SSL after GitHub issues the certificate.
Proxied orange-cloud CNAME sometimes fails GitHub's DNS check initially.


STEP 6 — Verify
---------------
1. Repo → Settings → Pages → Custom domain shows green checkmark
2. "Enforce HTTPS" becomes available — turn it ON
3. Open https://www.crimsonproject.com on phone
4. Add to Home Screen (Safari Share menu / Chrome Install app)


Troubleshooting DNS check failure
---------------------------------
- CNAME value must be YOUR_GITHUB_USERNAME.github.io (no https://, no path)
- CNAME file in repo must exactly match: www.crimsonproject.com
- Wait up to 24–48 hours for propagation
- Remove old A/CNAME records pointing elsewhere
- Use https://www.whatsmydns.net to check CNAME for www.crimsonproject.com
